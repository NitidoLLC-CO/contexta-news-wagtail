# Phase 2I Current vs Reference Gap

This document compares the current live public frontend against the approved reference sheets in `E:\Development\contextra_website\contexctra branding`.

## Homepage

Reference structure:

- Header, ticker, and status strip form a compact global command shell.
- Hero is an integrated first-fold system: lead story copy, immersive central/right visual, right Live Signals rail, vertical index, micro metrics, and CTA.
- Top Stories, AI Models Tracker, and Watchlist are visible immediately after the hero inside the same dense upper dashboard rhythm.
- Lower grid combines Signal Map, Trending Signals, Analysis & Insights, Research Briefs, and North Signal Brief in a compact multi-column module system.
- Footer Feature Strip, App CTA, and footer are dense and integrated.

Current structure:

- Header/ticker/status strip exist and have the right color direction.
- Hero exists but reads as a large isolated panel rather than a full dashboard composition.
- Live Signals rail exists but does not have the same first-fold authority as the reference.
- Top Stories, tracker, watchlist, and lower modules exist but feel stacked and less integrated.
- Footer/app CTA exist but do not yet match the reference pacing and bottom composition.

Missing modules:

- Strong upper dashboard band where Top Stories, AI Models Tracker, and Watchlist align tightly.
- Reference-like phone/app visual in the App CTA.
- Strong footer feature strip rhythm.
- More varied story cards inside the first dashboard band.

Wrong ordering:

- The first fold spends too much visual weight on the hero slab.
- Intelligence widgets need to sit closer to Top Stories and not feel like later independent sections.

Density gap:

- Current spacing is too generous.
- Reference uses tighter panels, more rows visible, and more content per viewport.

Mobile gap:

- Mobile is usable, but it does not yet match the reference sequence closely enough.
- Hero, live metrics, top stories, and bottom nav need a more app-like compressed rhythm.

## News Listing Page

Reference structure:

- Should feel like a section command center, not a generic archive.
- Compact hero/header, featured story, live updates, score/trending rails, varied article grid, documents/research, analysis and event modules.

Current structure:

- `/news/` is styled as an intelligence feed, but it still carries too much large-section-panel behavior.
- Featured story and rail modules are present but not compressed enough into the first viewport.
- Cards are improved, but the page still feels less dense and less modular than the reference.

Missing modules:

- Stronger right rail early in the first viewport.
- More reference-like grouping of trackers, documents, analysis, and related signals.
- Compact lower dashboard modules.

Wrong first fold:

- Page still opens with a larger command zone than the reference should require.
- Featured story should arrive earlier and feel more connected to the command header.

Repeated visuals issue:

- Branded North Signal fallback imagery appears too often as the dominant visual.
- Need more module-specific visual treatments for map, tracker, article cards, and section identities.

Right rail gap:

- Right rail does not feel as strong or as information-dense as the reference annotations.

Mobile gap:

- Mobile listing should show title, filters, featured story, and first cards quickly.
- Current mobile is clean but not close enough to the app-like reference hierarchy.

## Section Pages

Reference structure:

- Section pages should use a compact command header with title, subtitle, metrics, visual identity, and live updates.
- Featured story must appear in the first viewport.
- Right rail should include live updates, signal score, trending topics, and section-specific modules.
- Main feed should be varied and compact.
- Section-specific modules should reinforce each section, not just change copy.

Current structure:

- Phase 2G added section-specific labels and fallback content.
- Pages still start with large intro panels.
- Featured story appears too low relative to the reference.
- Section pages still feel repetitive because the layout and visual rhythm are too similar.

Missing modules:

- Compact first-fold featured story.
- Strong section-specific right rail in the first viewport.
- Better varied card rhythm in the feed.
- More visual differentiation beyond labels and small glyph variations.

Wrong first fold:

- Too much intro-only space.
- Not enough story/module density visible immediately.

Repeated visuals issue:

- Same North Signal visual still dominates.
- Need section/module-specific data visuals: compute grid, policy pillars, market chart, research graph, company network.

Right rail gap:

- Current right rail is helpful but weak compared with the desktop section/category reference.

Mobile gap:

- Section mobile first viewport shows title and live updates, but featured story should appear sooner.
- Chips and filters should feel closer to mobile reference smart filters.

## Article Page

Reference structure:

- Article hero includes breadcrumb, category, headline, dek, byline, date, read time, signal badges, visual media, and actions.
- Left context rail includes Article Context, radar/context diagram, Key Takeaways, and Related Topics.
- Main body has readable article typography, intelligence note, pull quote, and inline data card.
- Right rail includes Article Intelligence, Market Snapshot, Related Signals, Author, and Tags.
- Related story card/strip appears near bottom.

Current structure:

- Article page already has many of these modules from Phase 2B/2E/2H.
- It is closer than the section pages, but it still does not fully match the approved article reference.
- The composition needs tightening so headline, visual, actions, and context modules feel like one premium article briefing system.

Missing modules:

- Stronger `Next Story`/context micro-link is optional but visible in reference direction.
- More compact first viewport alignment between headline and hero media.
- Related story strip can be made denser.

Readability and intelligence-module gap:

- Body readability is acceptable.
- Context modules need a more reference-like position and rhythm.
- Right rail is useful but can feel stacked rather than integrated.

Mobile gap:

- Headline and actions appear early, but the media/action/body rhythm should more closely match the mobile article reference.
- Bottom nav must not cover content.

## Global Shell

Reference structure:

- Header/nav/search is tight, premium, and command-center-like.
- Ticker and status strip are compact and continuous.
- Bottom mobile nav has Home, Sections, center mark, Watchlist, Profile.

Current structure:

- Shell is close in color and content but not as precise/dense as the reference.
- Search is present; utility actions can be improved without backend changes.
- Mobile bottom nav exists but still differs from the reference labels and behavior.

## Overall Gap

The current frontend has the design language but not the reference layout architecture.

Phase 2I should focus on:

- first-fold compression
- denser dashboard zones
- stronger right rails
- varied story card rhythm
- less repeated logo-only visual usage
- section pages with featured stories in the first viewport
- mobile app-like ordering
- preserving all dynamic Wagtail/snippet/editorial systems while changing structure
